# Compétences en migration de site {#site-migration}

Les compétences en migration de site permettent d’intégrer des sites web dans Edge Delivery Services à l’aide de l’IA et des Ingénieurs déployés (FDE) Adobe. Cette compétence automatise la migration du contenu et des styles, et prépare une implémentation Edge Delivery Services prête pour l’auteur que vous pouvez adapter davantage via la création de code AEM.

## Vue d’ensemble {#overview}

Les compétences en migration de site transforment un site ou une conception existants en projet Edge Delivery Services. Une migration utilisant la compétence possède les composants suivants.

* **Sources** : site web existant (tout CMS, y compris l’instance de publication AEM), système de conception ou simulations de conception (par exemple, Figma).
* **Sorties** : référentiel Edge Delivery Services prêt pour la production auquel des blocs et des styles sont appliqués.
* **Cibles** : toutes les plateformes et méthodes de création Edge Delivery Services sont prises en charge en tant que cibles
* **Modèle de fonctionnement** : fourni par les ingénieurs déployés à terme Adobe qui gèrent l’agent et guident la migration pour garantir la qualité et la gouvernance

## Fonctionnalités {#capabilities}

La compétence migration du site effectue les tâches de haut niveau suivantes :

1. **Migration de contenu** - Convertit les pages et les ressources en contenu Edge Delivery Services structuré à l’aide du contenu par défaut et des blocs alignés sur la bibliothèque de blocs sélectionnée.

2. **Application de style** - Permet de créer des styles pour Edge Delivery Services de l’une des manières suivantes :

   * **Migration des styles en l’état** à partir du site actuel
   * **Reconception** en utilisant les simulations fournies (par exemple, Figma)
   * **réutiliser des conceptions de bloc existantes** à partir d’une implémentation Edge Delivery Services existante ;

Les livrables standard sont les suivants :

* référentiel Edge Delivery Services (blocs, styles, configuration)
* Contenu et ressources migrés
* Configuration de création (création de documents ou éditeur universel)
* Environnements Edge Delivery Services (aperçu et actif) dans lesquels le site migré est déployé.

Le résultat est un site web prêt pour la production optimisé pour les performances (Core Web Vitals/Lighthouse), l’accessibilité et les mises en page réactives sur les points d’arrêt, entièrement modifiable via la création de documents ou l’éditeur universel.

## Limites {#limitations}

Les compétences en migration de site se concentrent sur le contenu et le style. Les cas d’utilisation suivants nécessitent un effort d’implémentation supplémentaire en plus des compétences de migration du site :

* Intégrations personnalisées (par exemple, commerce, CRM, connecteurs de recherche)
* Contenu ou rendu dynamique (par exemple, contenu chargé de manière asynchrone, applications d’une seule page ou frameworks JavaScript personnalisées)
* Développement de nouvelles fonctionnalités sans rapport avec l’expérience migrée

Si vous avez besoin de ces fonctionnalités, les services Adobe Consulting ou votre partenaire d’implémentation peuvent étendre le projet Edge Delivery diffusé.

## Modèle d’engagement {#engagement}

Les compétences en migration de site sont actuellement disponibles par le biais d’un engagement géré par les ingénieurs Adobe Forward Deployed. Chaque projet commence par un exercice de définition de la portée en fonction de la complexité de la source, des objectifs de conception, de l’échelle du contenu et des exigences de gouvernance.

Pour explorer une migration :

* Contactez votre représentant Adobe ou l’équipe de votre compte pour lancer la définition de la portée et la planification.
* Adobe confirme l’éligibilité, estime l’engagement et propose un plan de migration.
